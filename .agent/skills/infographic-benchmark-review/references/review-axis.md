# Review Axis

## 観測軸とスコアリング基準

各軸は 1-5 のスコアで評価する。

### Axis 1: headline specificity（見出し具体性）

| Score | 基準 |
|-------|------|
| 5 | テーマ固有名詞と具体的数値を含む。テーマを知らない人でも主張が分かる |
| 4 | テーマ固有名詞を含むが、数値は不足 |
| 3 | テーマとの関連は読み取れるが、汎用的な表現が混在 |
| 2 | テーマ固有性が弱い。「業務効率化」「コスト削減」レベル |
| 1 | 完全に汎用的。「概要」「まとめ」「推進計画」等 |

### Axis 2: KPI clarity（KPI明確性）

| Score | 基準 |
|-------|------|
| 5 | 全KPIで value/unit が正しく分離。数値が現実的で文脈がある |
| 4 | value/unit 分離は正しいが、一部の数値に文脈が不足 |
| 3 | 概ね正しいが、value にテキスト混入や unit 欠落が散見 |
| 2 | KPI はあるが value/unit の分離が不完全 |
| 1 | KPI が欠落、または意味のない数値の羅列 |

### Axis 3: evidence density（エビデンス密度）

| Score | 基準 |
|-------|------|
| 5 | 全データページに sourceNote あり。facts が具体的で因果関係を説明 |
| 4 | 大部分のページに sourceNote あり。facts は概ね具体的 |
| 3 | sourceNote が半分程度。facts に一般論が混在 |
| 2 | sourceNote がほぼなし。facts が抽象的 |
| 1 | 根拠のない主張のみ。「大幅な改善」「飛躍的な向上」レベル |

### Axis 4: generic smell（汎用臭）

| Score | 基準 |
|-------|------|
| 5 | 全スライドがテーマ依存。テーマを差し替えると内容が成立しない |
| 4 | 大部分がテーマ依存。1枚程度に汎用的な表現 |
| 3 | 半分がテーマ依存、半分が汎用的 |
| 2 | 大部分が汎用的。テーマ名を変えても成立する |
| 1 | 全スライドが汎用テンプレート。テーマとの関連が表面的 |

### Axis 5: text safety（テキスト安全性）

| Score | 基準 |
|-------|------|
| 5 | 全フィールドが文字数制限内。レイアウト崩れリスクなし |
| 4 | 1-2箇所で制限ギリギリだが超過なし |
| 3 | 数箇所で軽微な超過（1-2文字） |
| 2 | 複数箇所で超過。レイアウト崩れの可能性あり |
| 1 | 大幅な超過が多数。確実にレイアウトが崩れる |

## 総合スコア計算

```
totalScore = (headlineSpecificity + kpiClarity + evidenceDensity + genericSmell + textSafety) / 5
```

## 品質判定

| 総合スコア | 判定 |
|-----------|------|
| 4.0-5.0 | 合格 -- ビジネス資料として十分な品質 |
| 3.0-3.9 | 条件付き合格 -- 改善余地あり、minor修正で使用可能 |
| 2.0-2.9 | 不合格 -- プロンプト改善が必要 |
| 1.0-1.9 | 重大不合格 -- 根本的な見直しが必要 |

## ベンチマーク結果記録形式

```
| テーマ | headline | KPI | evidence | generic | text | 平均 |
|--------|----------|-----|----------|---------|------|------|
| ERP刷新 | 4 | 3 | 4 | 4 | 5 | 4.0 |
| 営業DX | 3 | 3 | 2 | 2 | 4 | 2.8 |
| ...    | ...      | ... | ...      | ...     | ...  | ...  |
```

改善前後で同じ形式のテーブルを並べ、差分を可視化する。
