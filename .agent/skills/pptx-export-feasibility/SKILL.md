# pptx-export-feasibility

## Purpose

Assess and document PPTX export feasibility truthfully. Prevent overpromising editable PowerPoint output when the system generates image-based slides.

## Workflow

1. **Audit current export surface** -- enumerate existing export routes and their output formats
2. **Distinguish image-in-pptx vs native editable** -- clarify the difference between embedding images in .pptx containers and producing native editable PowerPoint content
3. **Define feasible scope** -- identify what can realistically be implemented in the short term
4. **Document fallback** -- describe the image-in-pptx approach as the only near-term option and label it truthfully

## References

- `references/native-vs-image-pptx.md` -- comparison of image-in-pptx vs native editable
- `references/export-surface-audit.md` -- current export routes and formats
- `references/file-map.md` -- export-related source files

## Key Principle

Never label image-in-pptx as "editable PowerPoint". Use "PowerPoint形式（画像版）" or similar honest labeling.
