# Business Slide Review Rubric

## 評価対象

ビジネスでそのまま使えるホワイトペーパー / ブリーフとしての品質。

## 評価基準

### 1. Cover Page (表紙)

#### 1.1 Thesis Clarity (主張の明確さ)

| Score | Description |
|-------|-------------|
| Excellent (4) | メイン主張が一瞬で伝わる。headline が目的を明確にしている。 |
| Good (3) | 主張は明確だが、さらに強められる余地がある。 |
| Acceptable (2) | 主張は何となく伝わるが、やや曖昧。 |
| Needs Improvement (1) | 何の資料か分からない。 |

#### 1.2 Support Presence (サブヘッドライン / KPI の存在)

| Score | Description |
|-------|-------------|
| Excellent (4) | Subheadline または KPI が主張を強く補強している。 |
| Good (3) | Subheadline があるが、より説得力を増やせる余地がある。 |
| Acceptable (2) | 最小限のサポートがある。 |
| Needs Improvement (1) | サポートが全くない。 |

#### 1.3 Visual Balance (ビジュアルバランス)

| Score | Description |
|-------|-------------|
| Excellent (4) | プロフェッショナルでクリーン。余白と要素配置が最適。 |
| Good (3) | 全体的にバランスが良い。軽微な調整で完璧。 |
| Acceptable (2) | 多少のバランス問題があるが、読みに支障なし。 |
| Needs Improvement (1) | 要素配置や余白に問題あり。 |

#### 1.4 Business Feel (ビジネス感)

| Score | Description |
|-------|-------------|
| Excellent (4) | 役員会でそのまま使えるレベル。装飾が控えめ。 |
| Good (3) | プロフェッショナル。軽微な infographic 感は残る。 |
| Acceptable (2) | 全体的にビジネス寄りだが、一部装飾が強い。 |
| Needs Improvement (1) | Infographic 感が強すぎる。 |

### 2. Executive Summary (エグゼクティブサマリー)

#### 2.1 Takeaway-First (要点先出し)

| Score | Description |
|-------|-------------|
| Excellent (4) | Key insights が最初に目に入る。takeaway が明確。 |
| Good (3) | Takeaway があるが、より強調できる。 |
| Acceptable (2) | 要点はあるが、探さないと見えない。 |
| Needs Improvement (1) | Takeaway がない。 |

#### 2.2 Data Support (データによる裏付け)

| Score | Description |
|-------|-------------|
| Excellent (4) | KPI や数値が要點を強く裏付けている。 |
| Good (3) | いくつかの数値があるが、もっと使える。 |
| Acceptable (2) | 最小限のデータがある。 |
| Needs Improvement (1) | データがない。 |

#### 2.3 Actionable (実行可能性)

| Score | Description |
|-------|-------------|
| Excellent (4) | 次にすべきことが非常に明確。アクション項目が具体的。 |
| Good (3) | 次のステップはあるが、より具体的にできる。 |
| Acceptable (2) | 行動は示唆されているが、やや曖昧。 |
| Needs Improvement (1) | 次のアクションがない。 |

#### 2.4 Structure (構造)

| Score | Description |
|-------|-------------|
| Excellent (4) | 非常に整理されている。論理的な流れが明確。 |
| Good (3) | 全体的に整理されている。軽微な混乱がある。 |
| Acceptable (2) | 構造はあるが、やや混乱気味。 |
| Needs Improvement (1) | 構造が混乱している。 |

### 3. Comparison (比較)

#### 3.1 Row Adequacy (行の十分性)

| Score | Description |
|-------|-------------|
| Excellent (4) | 4+ 行、構造が明確で意味のある比較。 |
| Good (3) | 3 行、十分な比較。 |
| Acceptable (2) | 2 行、最低限の比較。 |
| Needs Improvement (1) | < 2 行、比較として不十分。 |

#### 3.2 Clarity (明確さ)

| Score | Description |
|-------|-------------|
| Excellent (4) | 現状と推奨の区別が非常に明確。 |
| Good (3) | 区別は明確だが、表現に改善余地あり。 |
| Acceptable (2) | 区別はつくが、説明が必要。 |
| Needs Improvement (1) | 区別が不明確。 |

#### 3.3 Decision Support (意思決定支援)

| Score | Description |
|-------|-------------|
| Excellent (4) | 推奨が非常に説得力がある。 |
| Good (3) | 推奨はあるが、より強調できる。 |
| Acceptable (2) | 推奨があるが、やや弱い。 |
| Needs Improvement (1) | 推奨がない。 |

### 4. Roadmap / Decision-CTA (ロードマップ / 意思決定アクション)

#### 4.1 Phases (フェーズ)

| Score | Description |
|-------|-------------|
| Excellent (4) | 4+ フェーズ、各フェーズが明確に定義されている。 |
| Good (3) | 3 フェーズ、定義は良い。 |
| Acceptable (2) | 2 フェーズ、最低限。 |
| Needs Improvement (1) | < 2 フェーズ、不十分。 |

#### 4.2 Actionability (実行可能性)

| Score | Description |
|-------|-------------|
| Excellent (4) | アクションが非常に具体的で、誰がやるか明確。 |
| Good (3) | アクションは具体的だが、より詳細にできる。 |
| Acceptable (2) | アクションはあるが、やや曖昧。 |
| Needs Improvement (1) | アクションがない。 |

#### 4.3 Timeline (タイムライン)

| Score | Description |
|-------|-------------|
| Excellent (4) | 明確なタイムラインがある。 |
| Good (3) | タイムラインの言及がある。 |
| Acceptable (2) | 期間の言及はあるが曖昧。 |
| Needs Improvement (1) | タイムラインがない。 |

## Scoring Method

各カテゴリのスコアを平均化:

```
Cover Score = (1.1 + 1.2 + 1.3 + 1.4) / 4
Executive Summary Score = (2.1 + 2.2 + 2.3 + 2.4) / 4
Comparison Score = (3.1 + 3.2 + 3.3) / 3
Roadmap/Decision Score = (4.1 + 4.2 + 4.3) / 3

Overall Score = (Cover × 0.25) + (Executive Summary × 0.25) +
               (Comparison × 0.30) + (Roadmap/Decision × 0.20)
```

## Grade Mapping

| Overall Score | Grade | Action |
|--------------|-------|--------|
| 90-100 | Excellent | Business-ready。採用可。 |
| 80-89 | Good | 軽微な調整で完璧。 |
| 70-79 | Acceptable | 中程度の改善が必要。 |
| < 70 | Needs Improvement | 大幅な見直しが必要。 |

## Review Checklist

実装コードをレビューする際のチェックリスト:

### Cover Page

- [ ] Headline が short かつ clear
- [ ] Subheadline または KPI がある
- [ ] Visual balance が良好
- [ ] Business feel がある

### Executive Summary

- [ ] Takeaways が明確
- [ ] データによる裏付けがある
- [ ] Action items がある
- [ ] 構造が整理されている

### Comparison

- [ ] 3列構造 (項目|現状|推奨)
- [ ] 少なくとも 2 行
- [ ] 区別が明確
- [ ] 推奨がある

### Roadmap / Decision-CTA

- [ ] 3+ フェーズまたはアクション項目
- [ ] タムラインがある
- [ ] 次のステップが明確

### Overall

- [ ] スライド間の一貫性がある
- [ ] Business presentability が高い
- [ ] Generic infographic 感が弱い
- [ ] 読者が理解しやすい
